data-animation.js

Scroll ve load tabanlı animasyon yöneticisi. data-animation attribute ile elemanları görünür olduğunda .animate class'ı ekler — CSS transition'larınızla birlikte çalışır.

İçindekiler

  • on:load — Sayfa Yüklemede
  • on:scroll — Scroll Tetikleyici
  • threshold — Görünürlük Eşiği
  • reset:on-leave — Tekrar Gizle
  • media — Media Query Koşulu
  • CSS-Only Kullanım
  • Birleşik Örnekler
  • Parametre Referans Tablosu
  • Mimari ve Teknik Detaylar

on:load

Temel

Sayfa yüklendiğinde (window load event) .animate class'ını ekler. Hero bölümleri ve ilk ekranda görünecek elemanlar için idealdir.

data-animation="on:load"
🚀 Bu kutu sayfa yüklendiğinde fade-in ile belirir
HTML 
<div data-animation="on:load">
   Sayfa yüklendiğinde animasyon
</div>
Zamanlama: on:load, window load event'ine bağlıdır — tüm kaynaklar (resimler, fontlar) yüklendikten sonra çalışır. Hero alanı veya above-the-fold içerik için kullanın.

on:scroll

Temel

Eleman viewport'a girdiğinde .animate class'ını ekler. IntersectionObserver kullanır — performans dostudur.

data-animation="on:scroll"
✨ Bu kutu scroll ile görünür olduğunda animasyonla belirir
HTML 
<div data-animation="on:scroll">
   Scroll ile animasyon
</div>
Nasıl çalışır? on:scroll parametresi IntersectionObserver ile elemanı izler. Eleman viewport'a girdiğinde .animate class'ı eklenir. CSS ile istediğiniz geçiş efektini tanımlayabilirsiniz.

threshold

Ayar

Elemanın ne kadarının görünür olması gerektiğini belirler. 0–1 arasında değer alır (0.5 = %50 görünür olunca tetiklenir). Varsayılan: 0.01.

data-animation="on:scroll, threshold:0.3"
threshold:0.01 — Çok az göründüğünde tetiklenir (varsayılan)
threshold:0.3 — %30 göründüğünde tetiklenir
threshold:0.6 — %60 göründüğünde tetiklenir
HTML 
<!-- Varsayılan — neredeyse hemen tetiklenir -->
<div data-animation="on:scroll">...</div>

<!-- %30 göründüğünde -->
<div data-animation="on:scroll, threshold:0.3">...</div>

<!-- %60 göründüğünde -->
<div data-animation="on:scroll, threshold:0.6">...</div>
Gruplama: Aynı threshold değerine sahip elemanlar tek bir IntersectionObserver altında gruplanır. Farklı threshold değerleri ayrı observer'lar oluşturur — performans için minimum farklı threshold kullanın.

reset:on-leave

Davranış

Eleman viewport'tan çıktığında .animate class'ını kaldırır. Tekrar scroll edildiğinde animasyon yeniden çalışır — tekrarlayan efektler için kullanılır.

data-animation="on:scroll, reset:on-leave"
🔄 Reset aktif — Viewport'tan çıkınca gizlenir, tekrar girince animasyon oynar
🔒 Reset yok — Bir kez animasyon oynar, sonra kalır
HTML 
<!-- Her scroll'da tekrar animasyon -->
<div data-animation="on:scroll, reset:on-leave">
   Tekrarlayan animasyon
</div>

<!-- Yalnızca bir kez animasyon -->
<div data-animation="on:scroll">
   Tek seferlik animasyon
</div>
Performans: reset:on-leave IntersectionObserver'ı sürekli aktif tutar. Reset olmadan, eleman bir kez animate edildikten sonra observer devam eder ancak tekrar class eklemesi yapmaz.

media:(<query>)

Koşullu

Animasyonu bir media query koşuluna bağlar. Koşul eşleşmezse animasyon atlanır; koşul sonradan eşleşirse dinamik olarak devreye girer. Responsive animasyonlar için idealdir.

data-animation="on:scroll, media:(width < 1024px)"
📱 Sadece <1024px altında animasyon çalışır
🖥️ Sadece ≥1024px üzerinde animasyon çalışır
Pencere genişliği: px
HTML 
<!-- Sadece mobilde animasyon -->
<div data-animation="on:scroll, media:(width < 1024px)">
   Mobil animasyon
</div>

<!-- Sadece desktop'ta animasyon -->
<div data-animation="on:scroll, media:(width >= 1024px)">
   Desktop animasyon
</div>
Dinamik davranış: Media query eşleştiğinde animasyon aktifleşir, eşleşmezse .animate class'ı kaldırılır. Pencere boyutu değiştikçe anlık tepki verir (matchMedia change event).
Desteklenen format: width < 1024px, height >= 600px, width = 768px gibi kısaltılmış ifadeler. Script bunları otomatik olarak geçerli CSS media query'ye dönüştürür.

CSS-Only Kullanım

Bilgi

on parametresi olmadan kullanılan data-animation değerleri JavaScript tarafından sessizce atlanır. Bu değerler yalnızca CSS selector'lar için kullanılabilir — JS herhangi bir işlem yapmaz.

HTML 
<!-- JS tarafından atlanır — sadece CSS ile kullanılabilir -->
<div data-animation="reveal">CSS-only reveal</div>
<div data-animation="slide-up">CSS-only slide</div>
CSS Örneği 
/* CSS animation — JS müdahalesi olmadan */
[data-animation="reveal"] {
   animation: fadeIn 0.6s ease forwards;
}

[data-animation="slide-up"] {
   animation: slideUp 0.5s ease forwards;
}
Uyarı: threshold veya reset gibi mantıksal parametreler on olmadan kullanılırsa console'a uyarı yazılır. Ancak JS hiçbir class ekleme/kaldırma işlemi yapmaz.

Birleşik Örnekler

Pratik

Parametreler virgülle birleştirilerek güçlü animasyon davranışları oluşturulabilir. İşte gerçek dünyadan örnekler:

🎴 Sıralı Kart Animasyonu

Kart 1
Kart 2
Kart 3
HTML 
<!-- CSS transition-delay ile sıralı animasyon -->
<div data-animation="on:scroll, threshold:0.3"
     style="transition-delay: 0s">Kart 1</div>
<div data-animation="on:scroll, threshold:0.3"
     style="transition-delay: 0.15s">Kart 2</div>
<div data-animation="on:scroll, threshold:0.3"
     style="transition-delay: 0.3s">Kart 3</div>

🔄 Tekrarlayan Scroll Animasyonu

%50 göründüğünde kayarak gelir, viewport'tan çıkınca kayarak gider
HTML 
<!-- threshold + reset birlikte -->
<div data-animation="on:scroll, threshold:0.5, reset:on-leave">
   Tekrarlayan slide animasyonu
</div>

📱 Responsive Load Animasyonu

HTML 
<!-- Sadece mobilde load animasyonu -->
<div data-animation="on:load, media:(width < 768px)">
   Mobil hero animasyonu
</div>

<!-- Desktop'ta scroll, mobilde sadece CSS -->
<div data-animation="on:scroll, threshold:0.2, media:(width >= 1024px)">
   Desktop-only scroll animasyonu
</div>

✨ Tam Combo

🎯 threshold:0.4 + reset:on-leave — Scale efekti ile tekrarlayan animasyon
HTML 
<!-- Tüm parametreler birlikte -->
<div data-animation="on:scroll, threshold:0.4, reset:on-leave, media:(width >= 768px)">
   Desktop'ta, %40 göründüğünde, tekrarlayan animasyon
</div>
CSS Transition-Delay: Script sadece .animate class'ını ekler/kaldırır. Sıralı (stagger) animasyonlar için CSS transition-delay kullanın — JS tarafında ek yapılandırma gerekmez.

Mimari ve Teknik Detaylar

Çalışma Akışı 
  1.  dataAnimation() çağrılır
         │
  2.  document.querySelectorAll('[data-animation]') → tüm elemanlar
         │
  3.  parseAnimationData() → her elemanın config'i parse edilir
         │
  4.  media varsa → buildMediaQuery() + matchMedia dinleyici
         │
  5.  on:load → window load event'inde .animate eklenir
         │
  6.  on:scroll → elemanlar threshold'a göre gruplanır
         │
  7.  window load → her grup için IntersectionObserver oluşturulur
         │
  8.  Eleman viewport'a girince → .animate eklenir
         │
  9.  reset:on-leave → viewport'tan çıkınca .animate kaldırılır
JavaScript — Başlatma 
// ES Module import
import { dataAnimation } from './data-animation.js'

// Başlat — tüm [data-animation] elemanları initialize edilir
dataAnimation()
Observer Gruplama: Aynı threshold değerine sahip elemanlar tek bir IntersectionObserver altında gruplanır. Bu, çok sayıda animasyonlu elemanda bellek ve CPU kullanımını optimize eder.
Window Load: Scroll observer'ları window load event'inden sonra başlatılır. Bu, tüm DOM elemanlarının boyutlarının hesaplandığından emin olur (resim yükleme, layout shift önleme).
parseAnimationData(): key:value çiftlerini parse eder. Key-less değerler (örn. "reveal", "slide") sessizce atlanır — yalnızca CSS selector olarak kullanılabilirler. Geçersiz değerler console'a uyarı yazar.
buildMediaQuery(): width < 1024px gibi kısaltılmış ifadeleri (width < 1024px) CSS media query'sine dönüştürür. width ve height property'leri ile >, <, >=, <=, = operatörleri desteklenir.
